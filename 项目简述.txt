项目名称：MindPalace Agent (心智殿堂)
—— 你的私人认知进化实验室与多智能体（Multi-Agent）议事厅
1. 项目定位
MindPalace 不仅仅是一个信息聚合器，它是一个**“将信息转化为智慧”**的闭环系统。它通过主动抓取全网高质量内容，驱动由多个不同人格组成的 Agent 专家团，与用户进行深度对话、辩论和复盘，旨在通过“苏格拉底式提问”打破用户的信息茧房，提升心智深度。
2. 核心功能模块
A. 智能猎手 (The Scout) —— 精细化信息采编
功能： 区别于传统的 RSS 订阅，Agent 会根据用户的“认知图谱”在全网（微信公众号、知乎、X、学术论文、播客）筛选具有启发性的内容。
硬核点： 引入 “启发性评分机制”。Agent 会过滤掉情绪化煽动和碎片化垃圾，只推送能引发逻辑推演、底层原理思考或价值观碰撞的内容。
B. 智库议事厅 (The Council) —— 多智能体辩论系统
功能： 系统预设（或用户自定义）一组互补的人格模型，针对当日话题发起讨论：
理性批判者 (The Critic)： 负责寻找文章逻辑漏洞，挑战直觉思维。
跨界连接者 (The Synthesizer)： 负责将当前话题与历史知识或其他学科（如生物学、心理学）进行关联。
苏格拉底导师 (The Mentor)： 不直接给答案，而是通过连续追问引导用户自我表达。
交互： 用户置身于群聊中，可以作为观察者旁听 AI 之间的神仙打架，也可以作为参与者与他们公开辩论。
C. 认知账本 (Memory & Ledger) —— 长期记忆与心智评估
功能： 记录用户的观点演变。
硬核点： “回声定位”功能。当你对某个话题发表看法时，Agent 会调取你三个月前的类似言论进行对比：“你现在的观点比以前更务实了，是因为上次那个案例改变了你吗？”
3. 技术架构 (Technical Stack)
感知层： Firecrawl / Playwright (网页自动化解析) + LLM 自动化摘要与分类。
大脑层： 采用 LangGraph 框架。
为什么用 LangGraph： 因为你需要控制 Agent 之间的对话流（循环、分支、条件触发），LangGraph 比简单的 LangChain 更能处理复杂的“议事流程”。
记忆层： Mem0 (最新的 Agent 长期记忆框架) + Milvus/Pinecone (向量数据库)。用于存储用户的价值取向和知识储备。
模型层： GPT-4o (逻辑分析) + Claude 3.5 Sonnet (文学性与人文思考) + DeepSeek (极致性价比的中间处理)。
4. 项目亮点与含金量 (Selling Points)
从“工具”到“教练”的转变： 大多数项目是“搜索增强 (RAG)”，本项目是**“思考增强 (Thinking Augmented)”**。它挑战用户，而不是顺从用户。
复杂的状态管理： 在多智能体讨论中，如何防止对话陷入循环？如何确保不同性格的 Agent 保持人设一致性？这展示了开发者在 Prompt Engineering 和 Agent 编排 上的深厚功底。
主动式交互设计： 区别于“用户问-AI答”，本项目由 Agent 根据信息源主动发起对话，涉及对定时任务、异步处理和推送机制的综合运用。
